# 🎓 안드로이드 프로그래밍 기말 과제 발표 자료

## 1. 프로젝트 개요
- **프로젝트 명**: 유튜브 링크 관리자 (YouTube Link Manager)
- **개발 환경**: Java, Android Studio, Room Database
- **핵심 컨셉**: 나만의 유튜브 학습/취미 영상을 카테고리별로 분류하고 관리하는 개인용 아카이브 앱

---

## 2. 앱 개발 동기 (기획 의도)
- **파편화된 정보 관리**: 유튜브 앱 자체의 '나중에 볼 동영상'은 목록이 섞이기 쉽고 검색이 어렵습니다.
- **카테고리 중심의 학습**: 공부, 자기계발, 취미 등 주제별로 영상을 폴더 구조처럼 관리하여 필요할 때 즉시 찾아보고자 개발했습니다.
- **다크 테마 선호**: 영상 시청 환경에 맞춰 눈이 편안한 다크 모드 기반의 전용 UI를 구축하고자 했습니다.

---

## 3. 화면별 기능 소개

### ① 메인 화면 (MainActivity)
- **카테고리 폴더 UI**: 저장된 영상을 주제별로 묶어주는 카테고리 목록을 카드 뷰(CardView) 형태로 보여줍니다.
- **자동 초기화**: 앱 최초 실행 시 '공부', '개발' 등 기본 카테고리를 생성하여 편의성을 제공합니다.
- **추가 버튼**: 메인에서도 즉시 새로운 영상을 추가할 수 있는 플로팅 버튼(FAB)이 배치되어 있습니다.

### ② 영상 목록 화면 (VideoListActivity)
- **주제별 필터링**: 선택한 카테고리에 속한 영상만 리스트로 출력합니다.
- **유튜브 연동**: 아이템 클릭 시 `Intent`를 사용하여 외부 유튜브 앱이나 브라우저로 영상 링크를 즉시 전송 및 실행합니다.
- **상단 툴바**: 현재 보고 있는 카테고리 명을 툴바에 표시하고, 뒤로가기 기능을 제공합니다.

### ③ 영상 추가 및 수정 화면 (AddEditVideoActivity)
- **상세 정보 입력**: 영상 제목, URL, 메모를 입력할 수 있습니다.
- **유연한 카테고리 설정**: 스피너(Spinner)를 통해 기존 카테고리를 선택하거나, 새 카테고리를 즉석에서 추가할 수 있습니다.
- **데이터 관리**: 롱클릭을 통해 진입 시 기존 정보를 수정하거나 삭제할 수 있는 기능을 제공합니다.

---

## 4. 소스코드 주요 특징 (기술적 포인트)

- **데이터베이스 (Room Library)**:
    - `Category`와 `VideoItem` 두 개의 엔티티를 정의하여 관계형 데이터를 관리합니다.
    - 로컬 DB를 사용하여 인터넷 연결 없이도 저장된 목록을 즉시 확인할 수 있습니다.
- **UI/UX 디자인**:
    - **Material Design**: `CoordinatorLayout`, `FloatingActionButton`, `CardView` 등을 활용하여 안드로이드 표준 가이드라인을 준수했습니다.
    - **Custom Theme**: 유튜브의 정체성을 담은 Soft Red(`E53935`) 컬러를 주 색상으로 선정하고 다크 테마를 고정 적용했습니다.
- **어댑터 패턴**:
    - `CategoryAdapter`와 `VideoAdapter`를 통해 데이터와 뷰를 효율적으로 연결하고 메모리 사용을 최적화했습니다.

---

## 5. 프로젝트를 통해 배운 점
- **데이터 영속성**: Room DB를 다루며 데이터의 생성(C), 조회(R), 수정(U), 삭제(D) 흐름을 깊이 있게 이해했습니다.
- **화면 간 데이터 전달**: `Intent`와 `Extra`를 활용해 액티비티 간 데이터를 주고받는 실무적인 기술을 익혔습니다.
- **사용자 경험(UX)**: 다크 모드와 롱클릭 관리 기능 등을 구현하며 사용자 입장에서 편리한 인터페이스가 무엇인지 고민해보는 계기가 되었습니다.
